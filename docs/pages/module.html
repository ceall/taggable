Module

The Taggable Module provides a template tag which makes displaying your tags and associated data in your templates a breeze. There are a vast array of parameters you can use to filter, limit and alter your results, and lots of variables so you can get at the data you want. This tag functions as a system-wide outputter, as well as a tag cloud.

{exp:taggable}

Parameters

tag_id
Tag ID or IDs to lookup
tag_name
Tag name or names to lookup
tag_url_name
Tag URL name to lookup
url_separator
The URL separator to use when looking up by tag name and displaying a tag's URL name
entry_id
Finds all tags associated with a specific entry ID
entry_url_title
Finds all tags associated with a specific entry URL title
channel
Limit result set to specific channel or channels
sort
Sort results in ascending order or descending order
limit
Finds all tags associated with a specific entry ID
min_size
The minimum size in pixels
max_size
The maximum size in pixels
backspace
Strip the last X characters from the tag output

And you can use the following variables within that tag:

name
A tag name
id
A tag's numerical ID
description
A tag description
entry_count
The count of entries associated with a tag
url_name
A tag URL name
pretty_name
A tag name, formatted
size
The size of the tag in the cloud, in pixels

You can optionally specify some text, a redirect or something else to be outputted/executed when there are no tags. All you need to do is create a no_results conditional in your template, inside the {exp:taggable:tags} loop.

<ul>
	{exp:taggable entry_url_title="{segment_3}" limit="10"}
		<li>
			<h3><a href="{path=tags}/{url_name}">{name}</a></h3>
			<p>{description}</p>
		</li>
		
		{if no_results}
			<p>No tags!</p>
		{/if}
	{/exp:taggable:tags}
</ul>
