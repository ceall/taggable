Fieldtype

This section is all about the fieldtype itself - the heart and soul of Taggable.

Adding the Fieldtype

You can add the Taggable fieldtype to a custom field group like normal - in the Custom Field Administration section of the ExpressionEngine Admin Panel, add a new fieldtype and select "Taggable" from the drop-down menu. Make sure you make the field searchable to enable tag lookups. You can then use the custom field settings to customise how your fieldtype works.

Settings

The custom fieldtype settings panel allows you to customise the behaviour of your Taggable fieldtype. The following table explains each setting and what it does:

Setting Description
SAEF Field NameWhat field name to use for this field in the standalone entry form
SAEF SeparatorThe separator used to separate tags in a SAEF submission
Maximum tags per channel entryAllows you to limit the number of tags assigned to each entry. Set to 0 for unlimited.
URL SeparatorWhat to separate tags by in the URL. Used for lookups and tag output

The Template Tag

You can display the tags stored in the fieldtype in an easy way with a simple, but powerful template tag. This tag contains a range of variables that you can use to output your tags:

name
A tag name
id
A tag's numerical ID
description
A tag description
entry_count
The count of entries associated with a tag
url_name
A tag URL name
pretty_name
A tag name, formatted

The tag also supports an {if no_tags}{/if} conditional to optionally display content if no tags exist for this field.

Displaying Entries

Displaying entries using Taggable doesn't require any additional tags - you use the {exp:channel:entries} tag with the search: parameters. If you'd like to lookup entries tagged with a tag from the URL, it's as simple as passing through the value into {exp:channel:entries}. The following example assumes you want to lookup entries in a Taggable field named tags.

{exp:channel:entries channel="blog" search:tags="{segment_2}"}
	<h1>{title}</h1>
{/exp:channel:entries}

SAEF

Taggable has great support for ExpressionEngine's Standalone Entry Form. When you create your new custom field you specify the SAEF Field Name. To add SAEF functionality to your fieldtype, simply create a new HTML input or textarea element with the name of your specified SAEF Field Name. Taggable will then use your SAEF Separator to split the input into tags.
